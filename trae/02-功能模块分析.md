## 2. 逐模块分析

### 2.1 认证系统

**页面组成：**
- `/login` — 登录注册页（公开路由）

**API 状态：**
- 基于 Supabase Auth，无需自建 API
- 通过 `AuthProvider` 在全局管理认证状态

**前后端对接：** ✅ 已完整对接
- 登录/注册流程通过 Supabase 客户端 SDK 直接完成
- 会话管理由 Supabase 自动处理
- 全局 `AuthProvider` 负责用户状态分发

**数据库依赖：**
- `profiles` 表（7 字段）— 用户资料，AuthProvider 查询使用，侧边栏展示用户信息及权限判断

**结论：** 核心功能完整，可直接使用。

---

### 2.2 仪表盘

**页面组成：**
- `/` — 首页仪表盘（需登录）

**API 状态：**
- 无独立 API 路由，依赖其他模块的 API 获取数据

**前后端对接：** ❌ 使用 Mock 数据
- 前端页面已实现，但数据来源为 Mock
- 未对接记录、目标、分析等模块的真实 API

**数据库依赖：**
- 无直接数据库依赖，应从 `records`、`goals`、`analysis_history` 等表聚合数据

**待办事项：**

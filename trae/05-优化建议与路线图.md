## 2. P0 — 上线阻塞项（安全修复）

> **原则**：安全问题是上线的硬性门槛，必须在任何功能发布前全部解决。

### 2.1 配置 Supabase RLS（行级安全）

**问题描述**

当前数据库表未配置 Row Level Security (RLS) 策略。任何持有有效 Supabase 匿名密钥（anon key）的客户端，理论上可以读写其他用户的数据。这是一个**严重的安全漏洞**。

**影响评估**

| 维度 | 影响 |
|------|------|
| 安全性 | 🔴 **严重** — 数据泄露风险，用户隐私无法保障 |
| 合规性 | 🔴 **严重** — 违反数据最小权限原则 |
| 业务影响 | 🔴 **致命** — 上线即等于公开所有用户数据 |

**实施方案**

需要为以下表配置 RLS 策略：

| 表名 | 策略 | 说明 |
|------|------|------|
| `profiles` | SELECT：仅自己；UPDATE：仅自己 | 用户只能读写自己的档案 |
| `records` | CRUD：仅 owner | 成长记录完全私有 |
| `goals` | CRUD：仅 owner | 目标数据完全私有 |
| `supervision_relations` | SELECT：双方可见；INSERT/UPDATE：仅 supervisor | 督导关系双方可查看 |
| `reminder_rules` | CRUD：仅 owner | 提醒规则私有 |

**修复代码示例**

```sql
-- ============================================================
-- RLS 策略配置脚本
-- 执行顺序：在 Supabase Dashboard SQL Editor 中运行
-- ============================================================

-- 1. 启用 RLS
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE records ENABLE ROW LEVEL SECURITY;
ALTER TABLE goals ENABLE ROW LEVEL SECURITY;
ALTER TABLE supervision_relations ENABLE ROW LEVEL SECURITY;
ALTER TABLE reminder_rules ENABLE ROW LEVEL SECURITY;

-- 2. profiles 表策略
-- 用户只能查看自己的档案
CREATE POLICY "用户可查看自己的档案"
  ON profiles FOR SELECT
  USING (auth.uid() = id);

-- 用户只能更新自己的档案
CREATE POLICY "用户可更新自己的档案"
  ON profiles FOR UPDATE
  USING (auth.uid() = id);

-- 注册时自动创建档案（由触发器调用，使用 service_role）
CREATE POLICY "允许注册时创建档案"
  ON profiles FOR INSERT
  WITH CHECK (auth.uid() = id);

-- 3. records 表策略
-- 用户只能操作自己的记录
CREATE POLICY "用户可查看自己的记录"
  ON records FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "用户可创建自己的记录"
  ON records FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "用户可更新自己的记录"
  ON records FOR UPDATE
  USING (auth.uid() = user_id);

CREATE POLICY "用户可删除自己的记录"
  ON records FOR DELETE
  USING (auth.uid() = user_id);

-- 4. goals 表策略
CREATE POLICY "用户可查看自己的目标"
  ON goals FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "用户可创建自己的目标"
  ON goals FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "用户可更新自己的目标"
  ON goals FOR UPDATE
  USING (auth.uid() = user_id);

CREATE POLICY "用户可删除自己的目标"
  ON goals FOR DELETE
  USING (auth.uid() = user_id);

-- 5. supervision_relations 表策略
-- 督导关系双方可以查看
CREATE POLICY "督导双方可查看关系"
  ON supervision_relations FOR SELECT
  USING (
    auth.uid() = supervisor_id
    OR auth.uid() = supervisee_id
  );

-- 只有督导方可以创建关系
CREATE POLICY "督导方可创建关系"
  ON supervision_relations FOR INSERT
  WITH CHECK (auth.uid() = supervisor_id);

-- 只有督导方可以更新关系状态
CREATE POLICY "督导方可更新关系"
  ON supervision_relations FOR UPDATE
  USING (auth.uid() = supervisor_id);

-- 6. reminder_rules 表策略
CREATE POLICY "用户可查看自己的提醒规则"
  ON reminder_rules FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "用户可创建自己的提醒规则"
  ON reminder_rules FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "用户可更新自己的提醒规则"
  ON reminder_rules FOR UPDATE
  USING (auth.uid() = user_id);

CREATE POLICY "用户可删除自己的提醒规则"
  ON reminder_rules FOR DELETE
  USING (auth.uid() = user_id);

-- 7. Admin 策略（通过 user_metadata.role 判断）
-- 需要在 Supabase Auth 的 user_metadata 中设置 role: 'admin'
CREATE POLICY "管理员可访问所有记录"
  ON records FOR ALL
  USING (
    (auth.jwt() -> 'user_metadata' ->> 'role') = 'admin'
  );

CREATE POLICY "管理员可访问所有目标"
  ON goals FOR ALL
  USING (
    (auth.jwt() -> 'user_metadata' ->> 'role') = 'admin'
  );
```

### 2.2 添加输入验证（Zod）

**问题描述**

项目已安装 `zod` 依赖但未实际使用。所有 API Route Handler 接收请求体时未做任何验证，恶意用户可提交非法数据（如超长字符串、SQL 注入载荷、类型不匹配的字段等）直接写入数据库。

**影响评估**

| 维度 | 影响 |
|------|------|
| 安全性 | 🟠 **高** — 可能导致数据污染或注入攻击 |
| 稳定性 | 🟠 **高** — 未验证数据可能触发运行时错误 |
| 数据质量 | 🟡 **中** — 脏数据影响分析结果准确性 |

**实施方案**

**Step 1**：创建验证 Schema 目录 `src/lib/validations/`

**Step 2**：为每个 API 模块定义 Zod Schema

**Step 3**：在 API Route Handler 中统一使用验证中间件

**修复代码示例**

```typescript
// src/lib/validations/validate.ts
import { type NextRequest, NextResponse } from 'next/server';
import { type ZodSchema, ZodError } from 'zod';

/**
 * 验证请求体的工具函数
 * 在 API Route Handler 中使用，解析并验证 JSON 请求体
 */
export async function validateBody<T>(
  request: NextRequest,
  schema: ZodSchema<T>
): Promise<{ success: true; data: T } | { success: false; error: NextResponse }> {
  try {
    const body = await request.json();
    const data = schema.parse(body);
    return { success: true, data };
  } catch (err) {
    if (err instanceof ZodError) {
      return {
        success: false,
        error: NextResponse.json(
          {
            error: '请求参数验证失败',
            details: err.errors.map(e => ({
              field: e.path.join('.'),
              message: e.message,
            })),
          },
          { status: 400 }
        ),
      };
    }
    return {
      success: false,
      error: NextResponse.json(
        { error: '请求体格式错误' },
        { status: 400 }
      ),
    };
  }
}
```

### 2.3 错误信息脱敏

**问题描述**

当前 API Route Handler 在捕获异常时，可能直接将上游错误（如 Supabase 错误、OpenAI 错误）的完整信息返回给客户端。这些信息可能包含数据库连接字符串、表结构细节、API Key 前缀等敏感信息。

**影响评估**

| 维度 | 影响 |
|------|------|
| 安全性 | 🟠 **高** — 暴露系统内部架构和凭证信息 |
| 用户体验 | 🟡 **中** — 技术性错误信息对用户无意义 |

**实施方案**

建立统一的错误处理机制，生产环境只返回通用错误信息，详细错误记录到服务端日志。

**修复代码示例**

```typescript
// src/lib/errors.ts
import { NextResponse } from 'next/server';

/** 应用错误码枚举 */
export enum ErrorCode {
  BAD_REQUEST = 'BAD_REQUEST',
  UNAUTHORIZED = 'UNAUTHORIZED',
  FORBIDDEN = 'FORBIDDEN',
  NOT_FOUND = 'NOT_FOUND',
  RATE_LIMITED = 'RATE_LIMITED',
  INTERNAL = 'INTERNAL',
  EXTERNAL_API = 'EXTERNAL_API',
  DATABASE = 'DATABASE',
}

/** 应用自定义错误类 */
export class AppError extends Error {
  constructor(
    public code: ErrorCode,
    public statusCode: number,
    message: string,
    public details?: unknown
  ) {
    super(message);
    this.name = 'AppError';
  }
}

/** 预定义的常用错误 */
export const Errors = {
  unauthorized: () =>
    new AppError(ErrorCode.UNAUTHORIZED, 401, '请先登录'),
  forbidden: () =>
    new AppError(ErrorCode.FORBIDDEN, 403, '没有权限执行此操作'),
  notFound: (resource: string) =>
    new AppError(ErrorCode.NOT_FOUND, 404, `${resource}不存在`),
  badRequest: (message: string) =>
    new AppError(ErrorCode.BAD_REQUEST, 400, message),
  rateLimited: () =>
    new AppError(ErrorCode.RATE_LIMITED, 429, '请求过于频繁，请稍后再试'),
  internal: () =>
    new AppError(ErrorCode.INTERNAL, 500, '服务器内部错误，请稍后再试'),
  externalApi: () =>
    new AppError(ErrorCode.EXTERNAL_API, 502, '外部服务暂时不可用，请稍后再试'),
};

/**
 * 统一错误处理函数
 * 生产环境返回通用错误信息，开发环境返回详细错误
 */
export function handleApiError(error: unknown): NextResponse {
  console.error('[API Error]', {
    timestamp: new Date().toISOString(),
    error: error instanceof Error ? error.message : String(error),
    stack: error instanceof Error ? error.stack : undefined,
  });

  if (error instanceof AppError) {
    return NextResponse.json(
      {
        error: error.message,
        code: error.code,
      },
      { status: error.statusCode }
    );
  }

  // 生产环境不暴露未知错误的详情
  const isDev = process.env.NODE_ENV === 'development';
  return NextResponse.json(
    {
      error: isDev
        ? String(error)
        : '服务器内部错误，请稍后再试',
      code: ErrorCode.INTERNAL,
    },
    { status: 500 }
  );
}
```

### 2.4 添加 Rate Limiting

**问题描述**

当前所有 API 端点没有频率限制。恶意用户可以：
- 对登录接口进行暴力破解
- 对 AI 分析接口进行高频调用，消耗 OpenAI API 额度
- 对数据接口进行爬虫式批量抓取

**影响评估**

| 维度 | 影响 |
|------|------|
| 安全性 | 🟠 **高** — 暴力破解和 DDoS 风险 |
| 成本 | 🟠 **高** — OpenAI API 费用可能失控 |
| 可用性 | 🟡 **中** — 恶意流量可能影响正常用户体验 |

**实施方案**

**方案 A（推荐）**：使用 Upstash Redis + rate-limit 库

**方案 B（轻量）**：使用 Next.js Middleware + 内存计数器（适合原型阶段）

**修复代码示例**

```typescript
// src/lib/rate-limit.ts
import { NextRequest, NextResponse } from 'next/server';

/**
 * 基于内存的简单频率限制器
 * 适用于原型阶段，生产环境建议替换为 Upstash Redis
 */
const rateLimitMap = new Map<string, { count: number; resetTime: number }>();

interface RateLimitConfig {
  windowMs: number;  // 时间窗口（毫秒）
  max: number;       // 最大请求数
}

/** 预定义的限制策略 */
export const RateLimitPresets = {
  /** 登录接口：每 15 分钟最多 5 次 */
  auth: { windowMs: 15 * 60 * 1000, max: 5 },
  /** AI 分析接口：每分钟最多 10 次 */
  aiAnalysis: { windowMs: 60 * 1000, max: 10 },
  /** 普通 API：每分钟最多 60 次 */
  normal: { windowMs: 60 * 1000, max: 60 },
};

/**
 * 检查请求是否超过频率限制
 * @returns { limited: boolean, remaining: number, resetTime: number }
 */
export function checkRateLimit(
  identifier: string,
  config: RateLimitConfig
): { limited: boolean; remaining: number; resetTime: number } {
  const now = Date.now();
  const record = rateLimitMap.get(identifier);

  if (!record || now > record.resetTime) {
    rateLimitMap.set(identifier, {
      count: 1,
      resetTime: now + config.windowMs,
    });
    return { limited: false, remaining: config.max - 1, resetTime: now + config.windowMs };
  }

  if (record.count >= config.max) {
    return { limited: true, remaining: 0, resetTime: record.resetTime };
  }

  record.count++;
  return { limited: false, remaining: config.max - record.count, resetTime: record.resetTime };
}

/**
 * 获取客户端标识符
 * 优先使用 IP 地址，fallback 到 forwarded header
 */
export function getClientIdentifier(request: NextRequest): string {
  const forwarded = request.headers.get('x-forwarded-for');
  const ip = forwarded?.split(',')[0]?.trim() || '127.0.0.1';
  return ip;
}
```

```typescript
// src/middleware.ts
import { type NextRequest, NextResponse } from 'next/server';
import { checkRateLimit, getClientIdentifier, RateLimitPresets } from '@/lib/rate-limit';

/** Next.js 中间件：统一处理认证和频率限制 */
export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl;

  // 登录接口频率限制
  if (pathname.startsWith('/api/auth')) {
    const id = getClientIdentifier(request);
    const { limited, remaining, resetTime } = checkRateLimit(id, RateLimitPresets.auth);

    if (limited) {
      return NextResponse.json(
        { error: '请求过于频繁，请稍后再试' },
        {
          status: 429,
          headers: {
            'Retry-After': String(Math.ceil((resetTime - Date.now()) / 1000)),
            'X-RateLimit-Remaining': '0',
          },
        }
      );
    }
  }

  // AI 分析接口频率限制
  if (pathname.startsWith('/api/analysis')) {
    const id = getClientIdentifier(request);
    const { limited, remaining, resetTime } = checkRateLimit(id, RateLimitPresets.aiAnalysis);

    if (limited) {
      return NextResponse.json(
        { error: '分析请求过于频繁，请稍后再试' },
        { status: 429 }
      );
    }
  }

  return NextResponse.next();
}

export const config = {
  matcher: ['/api/:path*'],
};
```

```typescript
// src/hooks/use-api.ts
'use client';
import { useState, useEffect, useCallback } from 'react';

/** 通用 API 请求状态 */
interface ApiState<T> {
  data: T | null;
  loading: boolean;
  error: string | null;
}

/**
 * 通用 GET 请求 Hook
 * 封装 loading、error、data 状态管理
 */
export function useFetch<T>(url: string | null) {
  const [state, setState] = useState<ApiState<T>>({
    data: null,
    loading: true,
    error: null,
  });

  const refetch = useCallback(async () => {
    if (!url) return;
    setState(prev => ({ ...prev, loading: true, error: null }));
    try {
      const res = await fetch(url);
      if (!res.ok) {
        const body = await res.json();
        throw new Error(body.error || '请求失败');
      }
      const data = await res.json();
      setState({ data, loading: false, error: null });
    } catch (err) {
      setState({
        data: null,
        loading: false,
        error: err instanceof Error ? err.message : '未知错误',
      });
    }
  }, [url]);

  useEffect(() => {
    refetch();
  }, [refetch]);

  return { ...state, refetch };
}

/**
 * 通用 POST/PUT/DELETE 请求 Hook
 */
export function useMutation<T, B = unknown>(url: string, method = 'POST') {
  const [state, setState] = useState<ApiState<T>>({
    data: null,
    loading: false,
    error: null,
  });

  const mutate = useCallback(async (body: B) => {
    setState({ data: null, loading: true, error: null });
    try {
      const res = await fetch(url, {
        method,
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(body),
      });
      if (!res.ok) {
        const errBody = await res.json();
        throw new Error(errBody.error || '请求失败');
      }
      const data = await res.json();
      setState({ data, loading: false, error: null });
      return data;
    } catch (err) {
      const errorMessage = err instanceof Error ? err.message : '未知错误';
      setState({ data: null, loading: false, error: errorMessage });
      throw err;
    }
  }, [url, method]);

  return { ...state, mutate };
}
```

**服务端 SSE 端点**

```typescript

import { NextRequest } from 'next/server';
import OpenAI from 'openai';

const openai = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY,

/** AI 分析 SSE 流式端点 */
export async function POST(request: NextRequest) {
  const { records, goal, analysisType } = await request.json();

  const encoder = new TextEncoder();
  const stream = new ReadableStream({
    async start(controller) {
      try {
        const completion = await openai.chat.completions.create({
          model: 'gpt-4o-mini',
          messages: [
            { role: 'system', content: '你是一个专业的个人成长教练...' },
            { role: 'user', content: buildPrompt(records, goal, analysisType) },
          ],
          stream: true,
        });

        for await (const chunk of completion) {
          const content = chunk.choices[0]?.delta?.content || '';
          if (content) {
            controller.enqueue(
              encoder.encode(`data: ${JSON.stringify({ content })}\n\n`)
            );
          }
        }

        controller.enqueue(encoder.encode('data: [DONE]\n\n'));
      } catch (error) {
        controller.enqueue(
          encoder.encode(`data: ${JSON.stringify({ error: '分析失败' })}\n\n`)
        );
      } finally {
        controller.close();
      }
    },
  });

  return new Response(stream, {
    headers: {
      'Content-Type': 'text/event-stream',
      'Cache-Control': 'no-cache',
      Connection: 'keep-alive',
    },
  });
}
```

```typescript
// src/hooks/use-sse.ts
'use client';
import { useState, useCallback, useRef } from 'react';

/**
 * SSE 流式请求 Hook
 * 支持实时接收服务端推送的文本流
 */
export function useSSE() {
  const [content, setContent] = useState('');
  const [isStreaming, setIsStreaming] = useState(false);
  const abortRef = useRef<AbortController | null>(null);

  const startStream = useCallback(async (url: string, body: unknown) => {
    abortRef.current?.abort();
    abortRef.current = new AbortController();
    setContent('');
    setIsStreaming(true);

    try {
      const response = await fetch(url, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(body),
        signal: abortRef.current.signal,
      });

      const reader = response.body!.getReader();
      const decoder = new TextDecoder();

      while (true) {
        const { done, value } = await reader.read();
        if (done) break;

        const text = decoder.decode(value);
        const lines = text.split('\n').filter(line => line.startsWith('data: '));

        for (const line of lines) {
          const data = line.slice(6);
          if (data === '[DONE]') {
            setIsStreaming(false);
            return;
          }
          try {
            const parsed = JSON.parse(data);
            if (parsed.content) {
              setContent(prev => prev + parsed.content);
            }
          } catch { /* 忽略解析错误 */ }
        }
      }
    } catch (err) {
      if (err instanceof Error && err.name !== 'AbortError') {
        console.error('SSE error:', err);
      }
    } finally {
      setIsStreaming(false);
    }
  }, []);

  const stopStream = useCallback(() => {
    abortRef.current?.abort();
    setIsStreaming(false);
  }, []);

  return { content, isStreaming, startStream, stopStream };
}
```

```typescript
/** 数据导出 API 端点，支持 CSV 和 JSON 格式 */
export async function GET(request: NextRequest) {
  const user = await getCurrentUser(request);
  if (!user) {
    return NextResponse.json({ error: '请先登录' }, { status: 401 });
  }

  const { searchParams } = new URL(request.url);

    const csv = Papa.default.unparse(data);

      headers: {
        'Content-Type': 'text/csv; charset=utf-8',
        'Content-Disposition': `attachment; filename="${type}_export.csv"`,
      },
    });
  }

  if (format === 'pdf') {
    const { jsPDF } = await import('jspdf');
    const doc = new jsPDF();

    doc.setFontSize(18);
    doc.text(`GrowthMind - ${type === 'records' ? '成长记录' : '目标'}导出`, 14, 22);
    doc.setFontSize(10);
    doc.text(`导出时间: ${new Date().toLocaleString('zh-CN')}`, 14, 30);

    let y = 40;
    data.forEach((item: any, index: number) => {
      if (y > 270) {
        doc.addPage();
        y = 20;
      }
      doc.setFontSize(12);
      doc.text(`${index + 1}. ${item.title || '未命名'}`, 14, y);
      y += 7;
      if (item.content || item.description) {
        doc.setFontSize(10);
        const lines = doc.splitTextToSize(item.content || item.description, 180);
        doc.text(lines, 20, y);
        y += lines.length * 5 + 5;
      }
    });

    const pdfBuffer = doc.output('arraybuffer');
    return new Response(pdfBuffer, {
      headers: {
        'Content-Type': 'application/pdf',
        'Content-Disposition': `attachment; filename="${type}_export.pdf"`,
      },
    });
  }

  return NextResponse.json(data);
}

```

## 7. 风险评估

### 7.1 各阶段风险矩阵

| 阶段 | 风险 | 可能性 | 影响 | 缓解措施 |
|------|------|--------|------|---------|
| **Phase 1** | RLS 策略配置不当导致数据访问异常 | 中 | 高 | 编写详细测试用例，分表逐步启用 |
| **Phase 1** | Rate Limiting 影响正常用户体验 | 低 | 中 | 设置合理的限流阈值，添加白名单 |
| **Phase 2** | 数据库访问层迁移导致回归问题 | 中 | 高 | 保持双轨运行，逐步切换 |
| **Phase 2** | SSE 流式传输在生产环境不稳定 | 中 | 中 | 添加超时重试、断线续传机制 |
| **Phase 2** | 提醒引擎的定时任务可靠性 | 低 | 中 | 添加监控告警，失败重试机制 |
| **Phase 3** | 组件替换引入 UI 回归 | 高 | 中 | 逐步替换，每步截图对比验证 |
| **Phase 3** | 测试编写消耗过多时间 | 中 | 中 | 优先覆盖核心路径，TDD 迭代 |
| **Phase 4** | 国际化改造工作量超预期 | 中 | 低 | 分阶段上线，先支持中文 |

### 7.2 技术债务风险

| 债务类型 | 当前状态 | 风险说明 | 建议 |
|---------|---------|---------|------|
| 双重 ORM | Drizzle + Supabase SDK 并存 | 维护成本高，类型不一致 | Phase 2 统一 |
| Mock 数据 | 8 个页面使用 Mock | 隐藏真实 Bug | Phase 2 优先解决 |
| 无测试 | 测试覆盖率 0% | 重构无安全网 | Phase 3 补充 |
| 未使用依赖 | @aws-sdk 等 | 增加 bundle 大小 | Phase 4 清理 |

### 7.3 回滚策略

| 阶段 | 回滚方案 |
|------|---------|
| Phase 1 | RLS 策略可通过 `ALTER TABLE ... DISABLE ROW LEVEL SECURITY` 即时回滚 |
| Phase 2 | 保留 Feature Flag，可快速切换回 Mock 数据 |
| Phase 3 | 组件替换使用渐进式迁移，可逐个回滚 |
| Phase 4 | 增强功能独立模块，不影响核心流程 |

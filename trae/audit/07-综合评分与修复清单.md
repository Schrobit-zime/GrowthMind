### P0-01 `/api/profile` 越权访问漏洞（IDOR）

- **问题描述**：`/api/profile` 路由完全未调用 `authenticateRequest`，仅通过 `userId` 查询参数获取用户信息，任何人可通过 `GET /api/profile?userId=<任意UUID>` 获取任意用户的 profile（含 displayName、role、avatarUrl）。
- **影响范围**：所有用户资料可被越权读取，管理员角色信息泄露，可被用于后续攻击侦察。
- **文件路径和行号**：`src/app/api/profile/route.ts:10-31`
- **修复方案**：
  ```typescript
  // 修改前
  export async function GET(request: NextRequest) {
    const userId = request.nextUrl.searchParams.get("userId");
    const result = await db.select().from(profiles).where(eq(profiles.userId, userId))...
  }

  // 修改后
  import { authenticateRequest, unauthorizedResponse } from "@/lib/api-auth";
  import { handleApiError } from "@/lib/errors";

  export async function GET(request: NextRequest) {
    try {
      const auth = await authenticateRequest(request);
      if (!auth) return unauthorizedResponse();
      // 仅允许查询自己的 profile，或管理员查询他人
      const userId = request.nextUrl.searchParams.get("userId") || auth.user.id;
      if (userId !== auth.user.id) {
        // 需检查管理员权限
        const profile = await db.select().from(profiles)
          .where(eq(profiles.userId, auth.user.id)).limit(1);
        if (!profile[0] || profile[0].role !== "admin") {
          return NextResponse.json(
            { success: false, error: "无权访问他人资料" },
            { status: 403 }
          );
        }
      }
      const result = await db.select().from(profiles)
        .where(eq(profiles.userId, userId)).limit(1);
      return NextResponse.json({ success: true, data: result[0] || null });
    } catch (error) {
      return handleApiError(error, "获取用户资料");
    }
  }
  ```
- **预估工作量**：小

### P0-02 drizzle 直连 PostgreSQL 绕过 RLS

- **问题描述**：应用层数据查询使用 `drizzle-orm` 直连 PostgreSQL（`DATABASE_URL`），连接使用的数据库角色（postgres 超级用户或 service_role）**绕过 RLS**。即使 RLS 策略定义正确，所有通过 `db` 对象执行的查询均不受 RLS 保护。
- **影响范围**：所有数据库查询均无 RLS 保护，应用层遗漏 WHERE 条件即导致数据泄露。
- **文件路径和行号**：`src/lib/db.ts:6-17`
- **修复方案**：
  - **方案 A（推荐）**：所有用户数据查询改用 Supabase 客户端携带用户 JWT，使 RLS 生效
  - **方案 B**：在 drizzle 连接中设置 `request.jwt.claims`，模拟用户身份
  - **方案 C（短期）**：强化应用层 WHERE 条件审查，确保所有查询都包含 `userId` 过滤
- **预估工作量**：大

### P0-03 认证 Cookie 无 httpOnly/Secure 标志

- **问题描述**：所有认证 Cookie 通过 `document.cookie` 设置，无法设置 `httpOnly` 属性。JWT access_token 可被任意 JavaScript 读取，一旦发生 XSS 漏洞，攻击者可直接窃取 token。同时未设置 `Secure` 标志，HTTP 环境下 Cookie 会以明文传输。
- **影响范围**：所有登录用户的 access_token 可被 XSS 攻击窃取。
- **文件路径和行号**：
  - `src/components/auth/auth-provider.tsx:46, 48`
  - `src/app/(auth)/login/page.tsx:61, 76`
- **修复方案**：
  ```typescript
  // 改用服务端 cookies().set() 设置 Cookie
  // src/app/api/auth/callback/route.ts 或 login route handler
  import { cookies } from "next/headers";

  export async function setAuthCookie(token: string) {
    const cookieStore = await cookies();
    cookieStore.set("sb-access-token", token, {
      httpOnly: true,
      secure: process.env.NODE_ENV === "production",
      sameSite: "strict",
      path: "/",
      maxAge: 3600,
    });
  }
  ```
- **预估工作量**：中

- **影响范围**：管理员路由 `/supervise`、`/gateway` 的权限保护可被绕过。
- **文件路径和行号**：`src/middleware.ts:59-94`（特别是 72-75 行的 fetch 调用）
- **修复方案**：直接查询数据库或解码 JWT，不通过内部 fetch：
  ```typescript
  // 修改 middleware.ts 中的角色检查
  if (isAdminRoute(pathname) && accessToken) {
    try {
      // 直接查询 profiles 表获取角色
      const supabase = createClient(supabaseUrl, supabaseAnonKey, {
        global: { headers: { Authorization: `Bearer ${accessToken}` } },
      });

      if (!user) {

      const { data: profile } = await supabase
        .from("profiles")
        .select("role")
        .eq("user_id", user.id)
        .single();
      if (profile?.role !== "admin") {
        return NextResponse.redirect(new URL("/", request.url));
      }
    } catch {
      // catch 块应对 API 请求返回 401/403
      if (pathname.startsWith("/api/")) {
        return NextResponse.json(
          { success: false, error: "未授权" },
          { status: 401 }
        );
      }
      return redirectToLogin(request);
    }
  }
  ```

    const redis = getRedisClient();
    if (!redis) return;
    let cursor = "0";
    do {
      const [nextCursor, keys] = await redis.scan(
        cursor, "MATCH", pattern, "COUNT", 100
      );
      cursor = nextCursor;
      if (keys.length > 0) {
        await redis.del(...keys);
      }
    } while (cursor !== "0");
  }

  export async function applyRateLimit(
    request: NextRequest,
    config: RateLimitConfig
  ): Promise<boolean> {
    const redis = getRedisClient();
    if (!redis) return true; // Redis 不可用时放行

    const ip = getClientIp(request);
    const key = `ratelimit:${ip}:${config.windowMs}`;
    const count = await redis.incr(key);

    return count <= config.max;
  }

    const redis = getRedisClient();
    if (!redis) return null;
    const value = await redis.get(key);
    if (value === null) return null;
    if (value === "__NULL__") return null; // 空值缓存标记
    return JSON.parse(value);
  }

  export async function cacheSet<T>(
    key: string,
    data: T,
    ttl: number = 300
  ): Promise<void> {
    const redis = getRedisClient();
    if (!redis) return;
    const value = data === null ? "__NULL__" : JSON.stringify(data);
    const finalTtl = data === null ? 60 : ttl; // 空值缓存 60s
    await redis.set(key, value, "EX", finalTtl);

    handler: (request: NextRequest, auth: AuthResult, ctx: T) => Promise<NextResponse>

      } catch (error) {

      }

## 第八部分：审计结论

### 8.1 项目整体状况

GrowthMind 项目是一个基于 Next.js 16 + React 19 + Supabase + Drizzle ORM 的成长记录与分析应用，经过 6 个维度的专项审计，综合评分 **4.64/10**，处于**早期原型阶段**。

#### 优势

1. ✅ **技术栈现代**：Next.js 16 App Router + React 19 + Tailwind CSS 4 + shadcn/ui
2. ✅ **错误处理工具设计完善**：`errors.ts` 提供完整的错误码体系、状态码映射、生产环境脱敏
3. ✅ **校验层抽象良好**：`validations/validate.ts` 的 `validateBody` 函数统一了校验逻辑
4. ✅ **缓存层设计合理**：`cache.ts` 提供完整 API，Redis 集成完整
5. ✅ **TypeScript 严格模式**：项目零 @ts-ignore 使用
6. ✅ **ESLint 配置严格**：启用 `import/no-cycle` 规则
7. ✅ **资源清理规范**：所有定时器、事件监听器、AbortController 都正确清理
8. ✅ **响应格式基本统一**：`{ success: boolean, data?: T, error?: string }`
9. ✅ **敏感信息管理良好**：无硬编码密钥，环境变量使用规范
10. ✅ **SQL 注入防护完善**：全部使用参数化查询

#### 劣势

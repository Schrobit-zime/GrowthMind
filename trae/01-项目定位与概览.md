# GrowthMind 项目定位与概览

## 1. 项目定位

### 一句话定位

**记录成长，遇见更好的自己** — 一个将个人成长量化、可视化、并借助 AI 提供洞察的全栈 Web 应用。

### 核心价值主张

GrowthMind 不是简单的日记工具，而是一个**个人成长多维数据记录与智能分析平台**，专注于帮助用户系统化地追踪、分析和优化个人成长轨迹。

- **多维记录**：覆盖学习、工作、生活、身体、心情五大维度，全面捕捉个人成长数据
- **时间灵活**：支持日报/周报/月报/年报/早午晚报等多种时间粒度，适应不同记录习惯
- **AI 驱动**：通过统一模型接入层对接多家 LLM，生成趋势分析与优化建议
- **监督协作**：管理员可监督其他用户，设置智能提醒规则，支持团队成长管理

### 解决的用户痛点

1. **成长难以量化**：传统日记只能记录文字，无法量化学习进度、工作效率、健康状况等关键指标
2. **数据分散**：学习用 A 应用、工作用 B 工具、健康用 C 平台，数据无法关联分析
3. **缺乏洞察**：积累了大量数据却不知道如何分析，无法发现成长规律和改进方向
4. **坚持困难**：没有有效的监督和提醒机制，容易半途而废
5. **反馈滞后**：无法及时了解自己的成长趋势，错过最佳调整时机

## 2. 目标用户画像

### 核心用户群体

| 用户群体 | 典型特征 | 核心需求 |
|---------|---------|---------|
| **注重成长的职场人士** | 25-40 岁，有明确职业规划，希望持续提升技能 | 追踪工作产出、技能成长、职业发展目标达成情况 |
| **学生** | 18-25 岁，需要管理学业和课外活动 | 记录学习时长、考试成绩、课外活动参与情况 |
| **自由职业者** | 需要自律和自我管理，工作生活界限模糊 | 平衡工作与生活，追踪项目进度和个人状态 |
| **自我提升爱好者** | 对个人成长有强烈追求，喜欢数据驱动的自我管理 | 多维度量化个人状态，发现成长规律 |

### 典型使用场景

#### 场景一：职场人士的每日复盘

> 张伟是一名产品经理，每天下班后用 GrowthMind 记录：
> - **工作维度**：完成了 3 个需求评审，推进了 2 个项目进度
> - **学习维度**：阅读了《俞军产品方法论》第 5 章，学习时长 45 分钟
> - **身体维度**：步行 8000 步，睡眠 7 小时
> - **心情维度**：心情指数 8/10，因为项目获得认可而感到满足
>
> 周末查看 AI 分析报告，发现"学习时长与工作满意度正相关"，于是调整计划增加学习时间。

#### 场景二：学生的学期追踪

> 李华是一名大学生，使用 GrowthMind 追踪整个学期的成长：
> - 设定目标：GPA 3.5+、通过 CET-6、完成 2 个课程项目
> - 每日记录学习时长、作业完成情况、复习进度
> - 月度查看目标达成进度，及时调整学习策略
> - 学期结束时生成成长报告，总结经验教训

#### 场景三：自由职业者的自律管理

> 王芳是一名自由设计师，使用 GrowthMind 管理工作生活平衡：
> - 早晚报功能帮助规划和复盘每日工作
> - 追踪项目交付时间、客户满意度、收入变化
> - 监控久坐时长、运动频率、睡眠质量
> - AI 分析发现"连续工作超过 3 小时效率显著下降"，于是调整工作节奏

### 核心需求矩阵

| 需求层级 | 具体需求 | GrowthMind 解决方案 |
|---------|---------|-------------------|
| **基础需求** | 方便快捷地记录每日状态 | 多时间维度记录（日报/周报/月报），结构化表单 |
| **进阶需求** | 量化追踪目标进度 | 目标管理模块，自动计算 current_value 进度 |
| **高级需求** | 发现成长规律，获得优化建议 | AI 分析引擎，生成趋势报告和个性化建议 |
| **协作需求** | 获得外部监督和支持 | 监督关系管理，智能提醒规则 |

#### Server Components vs Client Components

| 类型 | 使用场景 | 优势 |
|------|---------|------|
| **Server Components (RSC)** | 数据获取、SEO 内容、静态展示 | 减少客户端 JS 包大小，更好的 SEO，更快的首屏加载 |
| **Client Components** | 交互逻辑、状态管理、浏览器 API | 支持 useState/useEffect，响应用户交互 |

**最佳实践**：
- 默认使用 Server Components，仅在需要交互时添加 `'use client'`
- 将 Client Components 推向组件树的叶子节点
- 使用 Server Actions 处理表单提交和数据变更

### 数据流

#### 数据写入流程

```
用户输入 → React Hook Form (验证) → API Route → Drizzle ORM → Supabase PostgreSQL
     ↓
   Zod Schema → 类型安全验证 → 错误处理 → 返回结果
```

#### 数据读取流程

```
页面加载 → Server Component → Drizzle ORM 查询 → 数据序列化 → 渲染 HTML
     ↓
客户端交互 → Client Component → fetch API → API Route → 数据返回 → 状态更新
```

#### AI 分析流程

```
用户触发分析 → API Route → 收集数据 → 构建 Prompt → 统一模型网关 → LLM 推理
     ↓
解析响应 → 结构化结果 → 存储 analysis_history → 返回前端 → Recharts 可视化
```

### 数据库 Schema

共 9 张核心表：

| 表名 | 用途 | 核心字段 |
|------|------|---------|
| **profiles** | 用户表 | id, user_id, display_name, role, avatar_url |
| **records** | 成长记录 | id, user_id, time_dimension, record_date, learning/work/life/health/mood (JSONB), mood_score, summary, goal_id |
| **goals** | 目标管理 | id, user_id, name, dimension, metric, target_value, current_value, deadline, status |
| **supervision_relations** | 监督关系 | id, admin_user_id, supervised_user_id, active |
| **reminder_rules** | 提醒规则 | id, admin_user_id, supervised_user_id, rule_type, condition, actions, enabled |
| **analysis_history** | 分析历史 | id, user_id, time_range, dimensions, analysis_type, result, model_used, tokens_used |
| **gateway_usage_log** | 网关用量 | id, user_id, provider, model, tokens_in, tokens_out, cost, source |
| **email_templates** | 邮件模板 | id, name, subject, body, variables |
| **health_check** | 健康检查 | id, updated_at |

#### 表关系

```
profiles (1) ──→ (N) records
profiles (1) ──→ (N) goals
records  (N) ──→ (1) goals (可选)
profiles (1) ──→ (N) supervision_relations (as admin)
profiles (1) ──→ (N) supervision_relations (as supervised)
profiles (1) ──→ (N) analysis_history
profiles (1) ──→ (N) gateway_usage_log
```

### GrowthMind 差异化定位总结

#### 核心差异化优势

1. **多维度 + 结构化**：不是通用笔记工具，而是专注个人成长的结构化数据平台
2. **AI 原生**：内置多模型 AI 分析，不是附加功能，而是核心能力
3. **时间灵活**：支持多种时间粒度，适应不同的记录习惯和复盘需求
4. **监督协作**：独特的监督功能，支持团队成长管理
5. **开源可控**：支持自托管，数据完全自控，隐私有保障

#### 目标市场定位

```
                    灵活性
                      ▲
                      │
           Notion ●   │
                      │
                      │   ● GrowthMind
                      │
        Daylio ●      │      ● Exist.io
                      │
                      └───────────────────► 专业性
```

GrowthMind 定位在**灵活性**和**专业性**的平衡点：
- 比 Notion 更专注，开箱即用
- 比 Daylio 更专业，多维度追踪
- 比 Exist.io 更可控，支持自托管

#### 竞争策略

1. **差异化竞争**：避免与 Notion 等全能工具正面竞争，专注个人成长垂直场景
2. **AI 驱动**：利用 AI 能力提供独特价值，生成个性化洞察和建议
3. **开源社区**：通过开源吸引开发者贡献，建立社区生态
4. **隐私优先**：强调数据隐私和自托管能力，吸引注重隐私的用户
